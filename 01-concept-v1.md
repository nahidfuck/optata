# Wishlist — Concept Doc v1.0

**Статус:** FROZEN (2026-07-11)
**Deadline v1.0:** 2026-08-01 — живий лінк, який можна кинути другу.
**Правило:** нічого з розділу «Backlog» не потрапляє в v1.0. Ніколи. Якщо з'являється нова ідея — вона йде в Backlog, а не в код.

---

## 1. Що це

Один вішліст на людину. Ти додаєш свої хотілки (фото + назва), кидаєш лінк на профіль — і замість нудного списку людина отримує **колоду карток**: одна за одною, рандомно, як тасування. Прогортав усі — розкривається повний профіль.

**Одне речення для лендінгу:** «Твої хотілки як колода карток, а не як таблиця в Google Docs.»

## 2. Кому і навіщо

**Основний сценарій:** ДН / свята. Людина кидає лінк друзям. Друзі відкривають, гортають, обирають, тихо бронюють. Власник не знає, що йому подарують — сюрприз живий.

**Вторинний:** самому собі — візуальний мудборд бажань.

**НЕ:** соцмережа, маркетплейс, discovery-платформа. Це v1.1+ (Explore).

## 3. Скоуп v1.0 — що Є

### Auth
- Реєстрація: email + password + username.
- Логін, логаут.
- Reset password через email (реальний лист).
- **Немає** email-верифікації. **Немає** OAuth.

### Профіль
- Один вішліст на юзера. Крапка.
- Username унікальний, `a-z0-9_`, 3–20 символів. URL: `/u/{username}`.
- Опційно: display name, коротке bio, аватарка.
- Профіль **публічний за замовчуванням**. Неавторизований бачить усе (крім лічильників переглядів).
- Пошук людей по username (prefix search) у хедері.

### Айтеми
- Максимум **40** на юзера. Ліміт жорсткий: кнопка «Додати» гасне на 40-му.
- Обов'язково: **назва**, **фото** (одне, завантаження вручну).
- Опційно: посилання, ціна + валюта (UAH / USD / EUR / PLN), нотатка.
- **Немає** категорій. **Немає** тегів. **Немає** сортування. Це навмисно — це каша з бажанок, і в цьому весь кайф.
- Порядок карток — тільки drag & drop власником. Гість бачить порядок власника.
- З фото автоматично витягується акцентний колір → фон картки.

### Shuffle (головна механіка)
- Заходиш на **чужий** профіль → відразу full-screen картка з рандомним предметом.
- Гортаєш → наступна. Колода **перетасована без повторів**.
- Прогортав усі АБО натиснув «Показати все» → колода їде вгору, під нею — повний профіль з grid.
- **Свій** профіль: shuffle теж є, але власник може вимкнути його назавжди (`localStorage`, кнопка «Не показувати мені shuffle»).
- Керування: свайп на мобілці, клік по картці / стрілки / пробіл на десктопі.
- Стан не запам'ятовується між сесіями. Кожен захід — нова колода.
- **0 айтемів:** shuffle не показуємо, одразу профіль з порожнім станом.
- **1 айтем:** shuffle не показуємо (колода з однієї картки — це смішно), одразу профіль.

### Бронювання
- Тільки **авторизований** може позначити «Я це дарую».
- Один айтем — одна бронь. Забронював — інші гості бачять «Уже дарують» (без імені).
- **Власник не бачить броні ніколи.** Ні прямо, ні опосередковано (див. tech-spec: API не віддає це поле власнику).
- Забронювати свій власний айтем не можна.
- Сторінка `/reservations` — список того, що ти забронював, щоб не забути.
- Власник видалив айтем → бронь тихо помирає, у бронюючого з'являється «предмет видалено».

### Перегляди
- Лічильник переглядів на айтемі. **Бачить тільки власник.**
- Перегляд = картка показана в shuffle АБО відкрито деталі айтема. Grid не рахується (інакше цифра безглузда).
- Унікально по сесії (анонімний cookie). Без ботозахисту — це іграшка, не аналітика.

### Лендінг
- Одна сторінка. Заголовок, живе демо shuffle (фейкові дані, працює одразу, без реєстрації), кнопка «Створити свій».

### Мова
- Мова всьго продукту - Англійська.

## 4. Скоуп v1.0 — чого НЕМА (свідомо викинуто)

| Викинуто | Причина |
|---|---|
| Explore / стрічка незнайомців | Cold start (3 юзери = мертвий екран) + модерація UGC. → v1.1 |
| Кілька вішлістів | Нуль цінності, +таблиця, +навігація, +порожні стани |
| Категорії / теги / сортування | Суперечить ідеї «каша з бажанок» |
| Парсинг OG-тегів з посилання | Rozetka/Amazon/Allegro ріжуть датацентрові IP (Render). Працюватиме на ~40% лінків = виглядатиме як баг. → v1.2 |
| Друзі, фоловінг | Не потрібно для основного сценарію |
| Коментарі | Модерація + спам. Не тягнеш це в безкоштовному пет-проєкті |
| Emoji-реакції | Замінені лічильником переглядів |
| Email-верифікація | Reset password покриває 95% болю |
| OAuth (Google) | +1 провайдер, +callback, +merge акаунтів |
| Dark theme | Подвійна робота з тінями/контрастом у neo-brutalism |
| Кілька фото на айтем | Ламає і grid, і shuffle |
| Конвертація валют | Дропдаун, і все |

## 5. Design direction

**Метафора:** цінник / подарунковий ярлик. Не «картка в застосунку», а **тег** — з пробитою діркою зверху, жорстким контуром і зсунутою тінню без блюру. Shuffle — це роздача тегів з колоди. Grid — це дошка, обвішана тегами.

**Стиль:** soft neo-brutalism. Жорсткий контур + зсунута тінь + великий радіус. Одна світла тема.

**Ключове обмеження:** UI — це сцена, а не актор. Колір дають **фото**. Хром має бути тихим, інакше 40 акцентних кольорів + яскравий інтерфейс = візуальний шум.

**Токени, шрифти, motion** → див. `02-tech-spec.md`, розділ 6.

## 6. Метрика успіху v1.0

Не MAU, не retention. Одна цифра:

> **5 живих людей створили вішліст мінімум з 5 айтемами, і хоча б 1 бронь від когось, хто не є тобою.**

Якщо це є до 1 серпня — v1.0 успішна, йдемо в v1.1. Якщо ні — проблема не в фічах, і додавання Explore нічого не врятує.

## Backlog (НЕ v1.0)

Сюди — все нове. Дописуй у кінець, не втручайся в скоуп вище.

### v1.1 — Explore
- Публічна стрічка рандомних айтемів незнайомців.
- Потребує: `users.is_discoverable` (default false, opt-in!), `item_reports` таблиця, адмінка-заглушка для банів, rate limit на аплоад.
- **Не вмикати, поки немає ≥50 юзерів** — інакше стрічка з 12 предметів по колу виглядає як мертвий продукт.
- Модерація: opt-in + репорти + ручний бан. Автомодерацію не робимо.

### v1.2 — Link enrichment
- Вставив лінк → бекенд тягне `og:image` / `og:title` / `product:price`.
- Обов'язково: SSRF-захист (блок приватних IP-діапазонів), таймаут 5с, ре-хостинг картинки у свій storage, fallback «не вийшло — завантаж вручну».
- Очікування: працює на ~40% магазинів. Це ОК, якщо fallback чесний.

### Далі (не пріоритизовано)
- Кілька вішлістів (ДН / загальний / «колись»).
- Підвищення ліміту 40 → 100 (спершу порахувати storage: 100 × 300KB = 30MB/юзер).
- Google OAuth.
- Dark theme.
- Приватний вішліст (доступ тільки за токеном-лінком).
- Експорт вішліста в картинку для сторіз.
- PWA / offline-перегляд.
- Нотифікація власнику «твій вішліст переглянули 12 разів» (email digest).
